Task: Architect and initialize a Python-based autonomous Job Application Agent.

Goal: Scrape Telegram channels for job postings, filter by specific criteria (Role, Experience, CTC), generate a tailored CV, and automate the application process via web portals or email.

1. Environment & Config:
Create a .env file template and a config.py to load: TELEGRAM_API_ID, TELEGRAM_API_HASH, BOT_TOKEN, and GEMINI_API_KEY.

2. Project Architecture & Modules:
Provide a modular file structure with the following components:

scraper.py: Use Telethon or Pyrogram to monitor specific Telegram channels. Filter messages for keywords (e.g., "SDE", "2+ years", "15 LPA") using Regex or LLM-based parsing.

processor.py: Use google-generativeai (Gemini) to parse unstructured text into a JSON format containing Role, Experience, CTC, and JD URL.

cv_engine.py: A module that takes a base profile.txt and a scraped JD, then uses Gemini to generate a tailored CV. Use ReportLab or FPDF (free) to output the CV as a PDF based on a simple LaTeX or Markdown template.

automator.py: Use Playwright (Chromium) for web-based applications (MyWorkday, Naukri) and smtplib for email-based applications.

human_loop.py: A utility to pause execution if a field in profile.txt is missing, prompt the user via terminal or Telegram bot, and update the .txt file.

tracker.py: Append application details (Date, Role, Link, CV_Path, Status) to a history.txt or tracker.csv.

3. Logic Flow:

Fetch message -> Filter -> Scrape JD -> Generate tailored CV -> check profile.txt for missing info -> Apply -> Log status.

4. Technology Stack (Free Tier focus):

Python 3.10+

Telethon (Telegram), Playwright (Web Automation), google-generativeai (LLM), pydantic (Data Validation).

Please start by generating the project folder structure and the scraper.py code to fetch and filter Telegram messages.

Pro-Tips for this Build:

Telegram Scraper: Use Telethon. It’s the most robust library for "user-bots" (which you'll need to read messages from channels you don't own).

The "Human in the Loop": I recommend using Pydantic models for your personal details. If a required field is None, trigger an input() function that writes back to your text file.

Web Automation: Playwright is generally better than Selenium for modern portals like MyWorkday because it handles "shadow DOM" elements and async loading much more reliably.

CV Generation: Instead of fighting with Word docs, have Gemini output Markdown or LaTeX, then use a Python library to convert that to PDF. It’s much cleaner and easier to automate.
